1. 介绍

优势

  • 大部分操作在本地完成
  • 完整性保证
  • 尽可能添加数据而不是删除或修改数据
  • 分支操作非常快捷流畅
  • 与 Linux 命令全面兼容

Git 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。

Git 与常用的版本控制工具 CVS, Subversion 等不同，它采用了分布式版本库的方式，不必服务器端软件支持。

3. Git 配置

配置个人的用户名称和电子邮件地址（这里的设置的签名和登录的远程仓库的账号、密码没有任何关系，只是为了区分是谁写的）：

  • 项目级别 / 仓库级别：仅在当前本地仓库范围有效（保存位置：./git/config 文件）
    $ git config user.name "rqq" 	# 用户名
$ git config user.email test@qq.com	# 邮箱
  • 系统用户级别：登录当前操作系统的用户范围（保存位置在：~/.gitconfig文件）
    $ git config --global user.name "rqq" 	# 用户名
$ git config --global user.email test@qq.com	# 邮箱

查看配置信息

要检查已有的配置信息，可以使用 git config –list 命令：

$ git config --global --list # 查看系统用户级别的配置
...
user.name=rqq
user.email=test@qq.com

$ git config --list		# 项目级别
...
user.name=rqq
user.email=test@qq.com

4. Git 基本操作

Git 的工作就是创建和保存你项目的快照及与之后的快照进行对比。

Git 常用的是以下 6 个命令：git clone、git push、git add 、git commit、git checkout、git pull，后面我们会详细介绍。

说明：

  • workspace：工作区
  • staging area：暂存区/缓存区
  • local repository：或本地仓库
  • remote repository：远程仓库

4.1创建本地仓库和提交

一个简单的操作步骤：

$ git init  # 本地库初始化
$ git add [file name]  #指定文件
$ git add . #所有
$ git commit -m "commit message" [file name] # 提交暂存区内容到仓库
  • git init - 初始化仓库。
  • git add . - 添加文件到暂存区。
  • git add [file name] -指定文件添加到暂存区
  • git commit - 将暂存区内容添加到仓库中

查看状态工作区、暂存区状态

$ git status

4.3 前进和后退

  • 基于索引值推荐

    $ git reset --hard 指针位置
# 例子：git reset --hard e851d39 #回到这个状态(hash值)

  • 使用 ^ 符号只能后退

    $ git reset --hard HEAD^
# 例子：git reset --hard HEAD^^
# 注意：几个 ^ 表示后退几步

  • 使用 ~ 符号只能后退

    $ git reset --hard HEAD~n # 一个 ^ 表示后退一步，n 个表示后退 n 步
# 例子：git reset --hard HEAD~3

reset 的三个参数比较

--soft 参数:
	- 仅仅本地仓库移动 HEAD 指针
--mixed 参数:
	- 在本地仓库移动 HEAD 指针
	- 重置暂存区
--hard 参数
	- 在本地仓库移动 HEAD 指针
	- 重置暂存区
	- 重置工作区

4.4 删除文件并找回

  • 前提：删除前，文件存存在时的状态提交到了本地库
  • 操作：git reset --hard [指针位置]
    • 删除操作已近提交到本地库：指针指向历史记录
    • 删除操作尚未提交到本地库：指针使用 HEAD

撤销错误添加到暂存区里的文件，可以输入以下命令：

$ git rm --cache 文件名

4.5 比较文件差异

  • 将工作区走中的文件和暂存区进行比较

    $ git diff [文件名]

  • 将工作区中的文件和本地库历史记录比较

$ git diff [本地仓库历史版本] [文件名]

  • 不带文件名比较多个文件

    $ git diff [本地仓库历史版本]

4.6 分支管理

什么是分支管理

  • 在版本控制过程中，使用多条线同时推动多个任务

分支的好处

  • 同时推进多个功能的开始，提高开发效率
  • 各个分支在开发过程中如果某一个分支开发失败，不会对其他分支有任何影响

分支操作

  • 创建分支

    ==git branch==

  • 查看分支

==git branch -v==

  • 切换分支

    ==git checkout [分支名]==

  • 合并分支

    • 第一步：切换到接受修改的分支（被合并，增加新内容）上

      git checkout[被合并的分支名]

    • 第二步：执行 merge

      ==git merge [有新内容的分支名]==

    • 或者

      $ git checkout -b 分支名 # 创建分支并直接切换到该分支

  • 删除分支

    $ git branch -d 分支名

  • 解决冲突(当两个人对都对这个文件有修改的时候，就需要手动解决了)

  • 第一步：编辑文件，删除特殊符号

  • 第二步：把文件修改到满意程度，保存退出

  • 第三步：git add [文件名]

  • 第四步：git commit -m “日志信息”(==注意：此时commit一定不能带文件名==)

5 将本地仓库推送到远程仓库(Github)

创建远程库地址别名

$ git remote -v # 查看远程地址别名
$ git remote add 别名(一般用origin) 远程地址

删除指定的远程仓库

$ git remote rm origin

修改远程仓库地址

git remote set-url origin <remote-url>

推送

$ git push 别名 分支名
$ git push -u 别名 分支名	# -u指定默认主机

克隆

$ git clone [远程地址]
  • 完整的把远程库下载到本地
  • 创建 origin 远程地址别名
  • 初始化本地库

拉取

$ git pull[远程仓库地址] [远程分支名]
$ git fetch[远程仓库地址别名] [远程分支名]
$ git merge[远程库地址别名/远程分支名]
  • pull = fetch + merge

解决冲突

经常团队开发的时候都会对远程仓库进行修改，如果两个都对该仓库有修改，会发生冲突，需要解决冲突。

  • 如果不是基于 GitHub 远程库的最新版本，不能推送，必须先拉取。
  • 拉取下来后如果进入冲突状态，则按照“分支冲突解决”操作解决即可。
